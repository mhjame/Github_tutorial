10 lệnh Git cơ bản mà Developer phải biết

Trong git, Repository (Repo) là 1 kho chứa, lưu trữ source code của bạn


có 2 loại repository: Local repository (Your own PC)
Remote respository (Github, GitLab, Bitbucket)

config rồi thì không cần config nữa
git config --global user.name "your_name"
git config --global user.email "your_email"
git config --list

1. Tạo repo

create a local repo in your own PC
git init

2. Link

link the local repository to an empty GitHub repository
git remote add origin [GitHub_Repo_URL]

3. Git Clone

Clone (tải) a remote Repo on GitHub to Local PC
git clone [GitHub_Repo_URL]

Git Workflow: Quy trình xử lý (Workflow) trên Git

Staging Area: khu chuẩn bị trc khi commit Comit lưu lại bản chụp những thay đổi -> comit vào local repo -> push lên remote repo trên git hub

4. Git Add

git add file

5. Git Commit

git commit -m "your_mesage"


thay đổi message dùng: git commit --amend -m "amend descriptive message"

6. Git Push

Uploads all local branch commits to GitHub

git push origin [branch-name]


eg: git push origin master

git add . (thêm tất cả file vào staging area)

git comit -m "Lai thay doi nè :))"

git comit --amend -m "change title comit"

git push origin master

7. Git status

git status


để biết được trạng thái của git của mình

git add .

git commit -m "your_message"

8. Git diff

to view the changes you made relative t0 staging area for the next commit

git diff id_comit1 idcomit_2

eg: git diff b325c da985

9. git log

Lists version history for the current branch

syntac: git log --oneline (hiển thị mỗi commit trên 1 dòng)

10. Git Reset

Remove the specified file form the staging area, but leave the working directory unchang

git reset
